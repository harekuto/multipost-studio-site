Privacy Policy

This policy explains how MultiPost Studio handles information when used as a local desktop application.

Information handled by the application

MultiPost Studio may handle account profile information returned by an authorized platform, OAuth access or refresh tokens, selected video files, titles, descriptions, tags, privacy choices, scheduling information, and upload status data.

How information is used

Information is used only to display connected accounts, prepare publishing jobs, send user-selected content and metadata to the selected platform, refresh authorization where permitted, and show upload results.

Account passwords

MultiPost Studio does not request or store TikTok, YouTube, Instagram, Google, or Meta account passwords. Authentication is completed on the platform’s official authorization page.

Local storage

Application settings, job history, and encrypted OAuth tokens are stored locally on the user’s computer. The user is responsible for protecting access to that computer and its files.

Sharing and sale of data

MultiPost Studio does not sell personal information. Content and metadata are transmitted only to services selected by the user as necessary to perform authorization or publishing actions.

Data retention and deletion

Local information remains until the user disconnects an account, deletes application data, or uninstalls the application. Users can also revoke access from the relevant platform’s account-security settings.

Third-party services

Use of TikTok, Google/YouTube, Meta/Instagram, GitHub, or other connected services is governed by those services’ own terms and privacy policies.

Security

Reasonable local protections are used, including encrypted token storage. No software can guarantee absolute security, and users should keep their operating system and application files protected.

Children

MultiPost Studio is not directed to children under the minimum age required to use the connected platforms.

Changes

This policy may be updated when the application’s features or data practices change.

Contact

Questions can be submitted through the project owner’s GitHub profile.
